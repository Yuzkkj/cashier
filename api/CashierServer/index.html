CashierServer

The server-side of Cashier. Handles item registration, purchase processing, gamepass tracking, gifting, inventory management, and stock-limited items.

Types​

ItemType​

</>
type ItemType = "DevProduct" | "Gamepass" | "Semipass"

Determines the type of an item.

ItemData​

</>
interface ItemData {
name: string--

The name of the item

type: ItemType--

The type of the item

id: number--

The id of the gamepass or developer product linked with the item

giftId: number?--

The id of the developer product for gifting this item

callback: ((
Player,
...any?
) → boolean)?--

A callback to be called when delivering the item to a player

predicate: ((
player: Player,
...any
) → boolean)?--

A function to check if the user can purchase the item with the given params

}

Configuration data for an item registered with Cashier.

local items = {
	{
		name = "VIP",
		type = "Gamepass",
		id = 1234,
		giftId = 4321,
		callback = function(player: Player)
			DataService:arrayInsert(player, "titles", "VIP")

			-- grants any extra benefits here

			return true
		end
	},
	{
		name = "Starter Pack",
		type = "Semipass",
		id = 7890,
		giftId = 987,
		callback = function(player: Player)
			DataService:update(player, "cash", function(value)
				return value + 5000
			end)

			return true
		end
	},
	{
		name = "Upgrade Unit",
		type "DevProduct",
		id = 6969,
		predicate = function(player: Player, unitId: string)
			-- make any validation here, for example
			-- if unitId is not nill
			-- if it is a valid unitId
			-- if it's not at max level, etc.

			return true
		end,
		callback = function(player: Player, unitId: string)
			-- the unitId will be the same one given to the predicate

			return true
		end
	},
}
WARNING

If the type of the item is set to "Gamepass", the id should also be a id of a gamepass.

</>
interface Item {

The id of the gamepass of developer product linked with the item

Useful information of an item.

GiftData​

</>
interface GiftData {
item: string--

The name of the item

from: number--

The id of the user that has gifted the item

}

Data from a gift.

PurchaseData​

</>
interface PurchaseData {
player: Player--

The player who bought the item

item: string--

The name of the item

robuxSpent: number--

The amout of robux spent on the purchase

recipientId: number?--

The id of the recipient if the purchase was an gift

}

Data from a purchase.

Limited​

</>
interface Limited {
name: string--

The name of the limited

id: number--

The developer product id linked to the limited

initialStock: number--

The initial stock when the limited was created

stockLeft: number--

The remaining stock available for purchase

onePerPlayer: boolean--

Determines if a player can only purchase this limited once

expiration: number?--

The Unix timestamp at which the limited expires

}

Configuration and state data for a stock-limited item.

type InitOptions = {ItemData}

Options for initializing the Cashier.

Functions​

init​

</>
CashierServer:init(options: InitOptions) → ()

Initializes the server-side of Cashier.

local DataService = require(path.to.DataService).server
local Cashier = require(path.to.Cashier).server

local items = require(path.to.items)
local limitedsCallback = require(path.to.limitedsCallback)

Cashier:init({
	itemsData = items,
	limitedsCallback = limitedsCallback
})
WARNING

:init(...) should be called before using any other method.

Cashier:addGift(6969, {
	item = "VIP",
	from = 9696
})

addLimited​

</>
CashierServer:addLimited(limited: Limited) → ()

Registers a new limited item. Writes it to the DataStore and notifies all connected clients.

local Cashier = require(path.to.Cashier).server

Cashier:addLimited({
	name = "Exclusive Sword",
	id = 123456,
	initialStock = 100,
	stockLeft = 100,
	onePerPlayer = true,
	expiration = os.time() + 86400,
})

addStockToLimited​

</>
CashierServer:addStockToLimited(
limitedName: string,
amount: number
) → ()

Adds stock to an existing limited item. Increments both stockLeft and initialStock.

local Cashier = require(path.to.Cashier).server

Cashier:addStockToLimited("Exclusive Sword", 50)

addTimeToLimited​

</>
CashierServer:addTimeToLimited(
limitedName: string,
amount: number
) → ()

Extends the expiration of a limited item by amount seconds. If the limited has no expiration set, it starts from the current time.

local Cashier = require(path.to.Cashier).server

-- Extend by 1 hour
Cashier:addTimeToLimited("Exclusive Sword", 3600)

removeLimited​

</>
CashierServer:removeLimited(limitedName: string) → ()

Removes a limited item from the DataStore and notifies all connected clients of the removal.

local Cashier = require(path.to.Cashier).server

Cashier:removeLimited("Exclusive Sword")

purchaseLimited​

</>
CashierServer:purchaseLimited(
player: Player,
limitedName: string
) → ()

Validates and prompts the player to purchase a limited item. Checks that the limited exists, has stock remaining, is not expired, and respects the one-per-player constraint. 

local Cashier = require(path.to.Cashier).server

local success = Cashier:purchaseLimited(player, "Exclusive Sword")
if not success then
	warn("Could not prompt limited purchase")
end

Checks if the player owns the given limited item.

local Cashier = require(path.to.Cashier).server

if Cashier:hasLimited(player, "Exclusive Sword") then
	print("Player already owns this limited!")
end
